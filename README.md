# SilentFlareGToS

**SilentFlareGToS** means **SilentFlare General Terms of Service**.

Markdown + GitHub Pages single-page policy document for the SilentFlare personal blog General Terms of Service.

This repository is for a personal blog / personal web presence. It is not a VPS, hosting, payment, SaaS, or commercial infrastructure service agreement unless SilentFlare publishes a separate written product-specific agreement.

## Structure

```text
GToS.md              # repository-readable canonical Markdown copy
docs/
  _config.yml        # GitHub Pages / Jekyll config
  index.md           # single-page GitHub Pages version
```

The public GitHub Pages site uses `docs/index.md` as the single web page.

## Deploy with GitHub Pages

Use GitHub Pages with Jekyll:

```text
Settings -> Pages -> Build and deployment
Source: Deploy from a branch
Branch: main
Folder: /docs
```

After deployment, the site entry is the Pages URL for this repository. The terms are now shown on one page instead of being split into multiple `/terms/.../` pages.

## Editing rules

- Use `SilentFlareGToS` as the canonical short name.
- Use `SilentFlare General Terms of Service` as the canonical full name.
- Write policy text in Markdown, not handwritten HTML.
- Keep the document suitable for a personal blog, not a commercial hosting platform.
- Edit `docs/index.md` when changing the public webpage.
- Keep `GToS.md` synchronized as the root repository copy when practical.
- Keep KYC1 limited and exceptional.
- Do not add KYC2, KYC3, proof-of-address, source-of-funds, or commercial onboarding unless the site changes into a commercial service.
- Avoid blanket protected-characteristic, ethnicity, nationality, religion, language, or continent-based restrictions. Controlled access should be based on objective legal, security, anti-abuse, crawler, provider, sanctions, or network-risk factors.
